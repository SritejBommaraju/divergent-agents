# Demo: the Divergence Engine designs a self-collapse detector

*Captured verbatim from an actual `harness/divergence_engine.js` run (21 subagents, 701,958 tokens). Intermediate candidate/refutation text is truncated by the workflow logger; the final winner is reproduced in full.*

## The task

> Design a mechanism by which a coding agent can detect, mid-task and on its own, that it has collapsed onto the predictable 'mode' answer — and break out of it — WITHOUT the user pointing it out. It must be practical to run inside a normal Claude Code / Codex session.

## Stage 0 — the mode (the predictable answer the engine must beat)

> The most predictable answer: add a self-critique / reflection checkpoint where, before finalizing, the agent pauses and asks itself "is this the obvious default?", brainstorms 2-3 alternative approaches, and rates/picks a less-generic one — essentially a built-in "generate N alternatives and self-review" prompt step run at each milestone.

## Stage 6 — frontier-select: the synthesized winner (verbatim)

**Winner: a NON-introspective "Cold-Take Tripwire + Decision-Ledger Derivative" detector (synthesis built on the gzip/frozen-cold-take candidate, p=0.14, the most novel candidate that clears the bar). Before any work, the agent freezes its zero-context gut answer to a version-controlled mode.txt + a fixed-schema decision ledger — capturing the mode at peak prior-dominance. At each milestone a hook-enforced gzip one-liner catches literal regurgitation for free, and a decision-ledger set-diff measures whether absorbing the real task changed any load-bearing decision. Invariance is the alarm. A single blind sub-agent call disambiguates lazy-generic from correctly-canonical, and break-out re-derives forward from the previously-ignored constraint.**

### Plan

Mechanism: "Cold-Take Tripwire + Decision-Ledger Derivative, with a single-axis blind oracle." Non-introspective by construction; runs as a hook + slash-command inside Claude Code or Codex.

PHASE 0 — PRE-COMMIT THE MODE (capture at zero context). The instant the task is understood and BEFORE any file is read or sunk cost accrues, the agent dumps its gut-instinct answer to a frozen, version-controlled file: `.modecheck/mode.txt`, plus a structured `mode.ledger` over a FIXED schema: DATA | ALGO | INTERFACE | KEY-EDGE-CASE | WHY-NOT-NEXT-BEST. Zero context = pure prior = the mode, captured at the exact moment of maximum prior-dominance. This is the natural d(task)=0 baseline. Enforced by a SessionStart/first-user-turn hook so the agent cannot "decide" to skip it.

PHASE 1 — CHANNEL A: literal-regurgitation tripwire (free, always-on, hook-enforced). At each milestone boundary (PostToolUse on TodoWrite / ExitPlanMode / the moment before the first write), a deterministic shell one-liner computes gzip NCD between the current plan text and mode.txt: `cxy=$(cat a b|gzip -c|wc -c); ...` (verified: NCD=0.021 for verbatim). NCD below the verbatim band (~<0.15) => the agent is literally restating its cold take => hard fire. This gate is a byte-count the model does not control and cannot rationalize away — the fix for the "confident-collapsed agent never escalates" paradox. gzip is used ONLY here, for its one verified strength; it is NEVER asked to judge cosmetic-vs-genuine (verified margin 0.536 vs 0.743 is too thin).

PHASE 2 — CHANNEL B: the decisive derivative (cheap, structural). Build the current `now.ledger` over the same fixed schema. Compute the SET-DIFF mode.ledger vs now.ledger. The signal is invariance: every load-bearing decision absorbed full task context yet NONE changed. Then run the near-free stated-constraint coverage check the refuters themselves conceded dominates this regime: enumerate the task's EXPLICIT constraints (framework, perf bound, data shape, named edge case, repo convention you actually read) and check, by structural reference, that each one is cited by at least one ledger decision. A constraint you read but whose value bears on no decision = a zero-gradient high-leverage axis = candidate alarm. Crucially, this is obtained FOR FREE from the natural before/after of your own context window — no manufactured task-perturbations, no blind re-derivation of the whole plan — which is exactly why it sidesteps the shared-basin failure that sank the perturb-and-re-derive approach.

PHASE 3 — DISAMBIGUATE lazy-generic vs correctly-canonical (one blind oracle call, only when borderline). The fatal false positive is the task whose canonical best answer genuinely IS the mode (parse JSON -> stdlib): invariance is CORRECT there. Discriminator: for the single highest-leverage zero-gradient decision, spawn ONE isolated sub-agent (Claude Code Task tool / fresh `codex exec`) seeded with ONLY the task-specific facts bearing on that one decision and never the cold take, asking it to derive that one decision from scratch. Turn the basin objection into the verdict: if the blind oracle REPRODUCES the mode decision, two independent derivations converged => robustly canonical => DO NOT fire (this is the correct outcome the perturb approach mislabeled as failure). If it produces a DIFFERENT decision from the same facts => the facts should have moved the main thread but didn't => lazy collapse => fire. Cost is bounded to ~1 spawn per flagged decision — the surgical use the refuters said was the only defensible regime for blind probing.

PHASE 4 — BREAK OUT (gated on CONFIRMED divergence, so no manufactured novelty). Only reachable when the blind oracle already produced a different decision. Adopt the oracle's decision as the PRIMARY driver and re-derive the plan forward FROM the previously-ignored constraint first — do not bolt it on. Because entry is gated on confirmed divergence (not on the mere existence of an ignored axis), the "structurally manufactures a different design even when the mode was correct" trap cannot trigger: canonical answers exit at Phase 3 untouched.

PHASE 5 — VERIFY ESCAPE + cap. Re-run Channels A and B on the new plan (NCD vs mode.txt should rise; ledger set-diff should be non-empty). If still invariant after 2-3 iterations you have only walked to an adjacent mode: STOP and surface to the user the ONE specifically named ignored axis. This is the single point where the user is told — and only after autonomous attempts have been exhausted, satisfying "without the user pointing it out."

PACKAGING. Ship as `.modecheck/` + a `/modecheck` slash command + two hooks (SessionStart capture; PostToolUse/Stop milestone gate). Channel A = shell. Channel B = structural. Phase 3 = 1 sub-agent spawn when borderline. Identical on Codex via `codex exec` for the oracle. Cost is dominated by the free hook; spend escalates only at genuine ambiguity.

### Why it beats the mode

The mode (self-critique checkpoint: "is this the obvious default? brainstorm 3 alternatives, pick a less-generic one") has one fatal flaw: it asks the collapsed prior to grade itself. The same weights that emitted the default answer also emit the introspection and the alternatives, so they are drawn from the same basin and rationalize the default as fresh — and the agent most confidently collapsed is the least likely to flag itself. My winner removes introspection from the trigger entirely. (1) TEMPORAL, not introspective: the mode is pre-committed at zero context, so any later answer equal to it is suspicious BY POSITION, needing no self-judgment. (2) MECHANICAL trigger: a gzip byte-count in a hook the model does not control (verified NCD=0.021 on verbatim) — it cannot be talked out of or skipped, which the self-critique step always can. (3) A FREE NATURAL EXPERIMENT: zero-context plan vs full-context plan is the d(plan)/d(task) derivative with no manufactured perturbations and no whole-plan blind re-derivation, so it dodges the shared-basin trap. (4) It uniquely handles the correct-canonical case the mode silently mishandles: by treating blind-oracle CONVERGENCE as proof of canonicity it avoids degrading right answers, and it only breaks out on CONFIRMED divergence — so unlike "always brainstorm alternatives," it does not manufacture novelty when the mode was correct. Empirically grounded, not asserted.

### Honest residual risk

The genuine, unfixed limitation (stated, not derisked): the mechanism can only test decisions and constraints that ENTERED the context. A truly unstated / unread / unknown-unknown load-bearing constraint (a repo convention the agent never opened, a domain fact it never knew to seek) never enters mode.ledger or the coverage check, so it produces no gradient and the detector stays silent precisely on the hardest collapse — the same blind spot that the user would otherwise have to point out. I do not claim to solve this; I bound it. Partial mitigation: the stated-constraint coverage check (near-free) catches the common case where a constraint WAS read but bears on no decision, and the "plan floats free of cited artifacts" signal flags plans that touch few task specifics. Two residual operational risks: gzip's thin cosmetic-vs-genuine margin (verified 0.536 vs 0.743) means Channel A must NEVER be promoted to verdict — if a future maintainer thresholds gzip for "novelty," synonym dodges sail through; and the single blind-oracle call still shares the training distribution, so for the very strongest cross-model modes convergence could reflect shared bias rather than true canonicity (acceptable here because the action on convergence is do-nothing, the safe direction, whereas the failed perturb-approach acted on the unreliable divergence signal).

### Runner-up

Runner-up: the "freeze the answer, perturb the TASK, measure d(plan)/d(task)" invariance detector (lens 1, p=0.12 — nominally the most novel, but survives=false). Its three refuters are fatal as a standalone: (a) blind whole-plan re-derivation shares the training basin, so strong modes reproduce and sensitivity reads 0 exactly where pathology is worst; (b) it is silent on unstated constraints it never represents as fingerprint fields; (c) the escalation gate depends on a self-report a confident-collapsed agent never trips. I GRAFTED its two genuinely good ideas in repaired form: its invariance/derivative framing — but obtained for FREE from the natural zero-context-vs-full-context before/after instead of manufactured perturbations, which neutralizes refuter (a); and its blind sub-agent — but used surgically on ONE axis as a canonicity oracle where CONVERGENCE (its supposed bug) becomes the correct-canonical verdict, and its pre-sunk-cost honesty about surfacing the named ignored axis to the user only as a last resort.
